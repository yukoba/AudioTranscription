# 開発仕様書：音声認識＆自動入力 常駐アプリ

## 1. プロジェクト概要
Windows 11環境で動作する、音声認識を活用した常駐型アプリケーションを開発する。
グローバルホットキーで録音の開始・終了を切り替え、Gemini API で文字起こしと「えっと」などのフィラー除去を一括で実行し、結果を直接文字入力する。

## 2. 技術スタック
- **ターゲットフレームワーク**: .NET 10 / WPF (Windows Presentation Foundation)
- **言語**: C#
- **主要ライブラリ (NuGet)**:
  - `NAudio`: マイク音声のキャプチャおよびWAVファイル生成
  - `NHotkey.Wpf`: グローバルホットキーの登録と検知
  - `Google.GenAI`: Gemini API通信
  - `Hardcodet.NotifyIcon.Wpf` (または標準の `NotifyIcon`): タスクトレイ（通知領域）への常駐
  - `Microsoft.Extensions.Configuration`: ユーザー設定（APIキー等）の保存と読み込み

## 3. 画面・UI仕様
本アプリはメインウィンドウを持たず、タスクトレイ常駐およびオーバーレイUIのみで動作する。

- **タスクトレイアイコン**:
  - アプリ起動時はタスクトレイにアイコンを表示し、バックグラウンドで待機する。
  - 右クリックメニュー（コンテキストメニュー）:
    - 「終了」: アプリを完全に終了する。
- **録音中オーバーレイ表示**:
  - 録音中であることを示すインジケーター（「🎤 録音中...」というテキスト）。
  - 仕様: 常に最前面（`Topmost="True"`）、背景透明（`AllowsTransparency="True"`）、枠なし（`WindowStyle="None"`）。画面の中央下部に配置し、マウスクリックを透過する（`IsHitTestVisible="False"`）。
  - 録音開始ホットキーで表示し、録音終了ホットキーで非表示にする。

## 4. 機能要件・動作フロー
1. **待機状態**:
   - アプリ起動後、グローバルホットキー（`Ctrl + Shift + Space`。設定で変更可能が望ましいが固定する）をシステムに登録し、待機する。
   - Gemini APIキーは環境変数 `GEMINI_API_KEY` もしくは `%USERPROFILE%\AudioTranscription.json` から読み込む。
2. **録音開始**:
   - ユーザーがホットキーを押下する。
   - 録音中オーバーレイUIを表示する。
   - `NAudio` を使用して、既定のマイクデバイスから音声のキャプチャを開始し、一時ファイルに保存する。
3. **録音終了と処理開始**:
   - ユーザーが再度同じホットキーを押下する。
   - 録音中オーバーレイUIの表示を「⏳ 文字起こし中...」に変更して、文字起こしが完了したら非表示にする。文字起こし中にホットキーが押されても無視する。
   - 音声のキャプチャを停止し、WAVデータ（16kHz, 16bit, モノラル）を保存後、MP3形式に圧縮する。
4. **音声認識とフィラー除去・テキスト整形**:
   - MP3データとプロンプトを Gemini API (`gemini-3-flash-preview`) に送信して、文字起こしとフィラー除去を一括で行う。
   - システムプロンプト: 「あなたは文字起こしアシスタントです。音声の内容を文字起こししてください。その際、「えっと」や「あの」などのフィラーは除去し、句読点を付けてください。整形後のテキストのみを出力し、解説や挨拶は含めないでください。音声に人間の発話が含まれない場合は、いかなる推測も行わず `@@@` のみを出力してください。」
   - Thinking level は minimal に設定し、温度は0に設定する。
5. **テキストの直接入力**:
   - 取得したテキストが `@@@` 以外なら SendKeys を用いてアクティブなウィンドウに直接入力する。
   - 処理完了をシステム音（`SystemSounds.Asterisk.Play()`など）で通知する。
